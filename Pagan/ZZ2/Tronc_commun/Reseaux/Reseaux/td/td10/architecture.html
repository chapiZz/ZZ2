PPP - architecture

PPP et les couches de la technologie TCP/IP

La couche PPP est une couche MAC, qui peut être utilisée en dessous d'une couche IP.

Les composants de PPP

PPP contient plusieurs services qui utilisent PPP qui encapsule les diverses PDU pour qu'elles puissent être transmises sur le ligne téléphonique.
PPP est spécifié dans les RFC 1661

  • PAP (PPP Authentification Protocol)
    CHAP (Challenge Authentification Protocol)

    rôle : authentifier l'appelant
    documentation : RFC 1334


  • NCP (Network Control protocol)

    rôles
    - configurer la couche Réseau
    - il existe plusieurs NCP parmi lesquels :
    IPCPpour IP (le plus utilisé)
    IPXCPpour IPX
    ATCPpour Appletalk
    OSINLCPpour OSI
    - Certains protocoles peuvent avoir des fonctions particulières.
    Par exemple, IPCP compresse les entêtes IP, TCP et UDP, ce qui permet de diminuer notablement la quantité d'octets tranmis.

    documentation sur IPCP : RFC1332 et RFC1877


  • encapsulation PPP

    rôle : transmettre de manière fiable les PDU des protocoles CHAP, PAP, LCP et NCP, en les multiplexant sur une liaison série
