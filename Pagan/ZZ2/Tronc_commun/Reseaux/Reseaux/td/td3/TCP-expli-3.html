Marie-Claude Vialatte

Contrôle de flux

Le contrôle de flux est le mécanisme par lequel un destinataire contrôle le rythme d'émission d'un expéditeur de données.
C'est pour éviter que des PDU de données soient envoyées alors que le destinataire n'est pas en état de les recevoir ou de mémoriser les octets dans son buffer de réception.

Le protocole TCP inclus un mécanisme de contrôle de flux :

  • Chaque entité-TCP qui émet une PDU-TCP indique dans le champ win combien il reste de place dans son buffer de réception.
    Elle annonce ainsi sa capacité de réception.

  • Une entité-TCP n'émet pas de PDU de données contenant plus d'octets de données qu'il n'y a de place dans le buffer de réception du destinataire.
